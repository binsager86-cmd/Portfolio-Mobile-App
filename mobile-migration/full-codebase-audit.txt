1. PROJECT OVERVIEW
================================================================================

A full-stack portfolio tracker originally built with Streamlit, migrated to a
FastAPI REST backend + React Native / Expo mobile frontend.

Primary markets: Kuwait Stock Exchange (KWD) and US equities (USD).

Key capabilities:
  * Multi-portfolio management (KFH, BBYN, USA groupings)
  * Real-time price updates via Yahoo Finance
  * AI-powered portfolio analysis (Google Gemini)
  * Fundamental analysis via PDF extraction (PyMuPDF + Gemini OCR)
  * Personal finance management (PFM) with budget tracking
  * Push notifications (Expo / FCM)
  * Trade signal generation (P/E trend, whale-candle detection via EODHD)
  * News feed from Boursa Kuwait announcements
  * Backup / restore (XLSX / JSON)
  * Admin console (user management, activity log)
  * Google Sign-In (OAuth2)
  * i18n (English + Arabic)

3.1  ENTRY POINT & LIFESPAN
----------------------------
File: backend-api/app/main.py

Framework: FastAPI with asynccontextmanager lifespan
Run:       uvicorn app.main:app --reload --port 8004

Startup sequence:
  1. Verify dev_portfolio.db exists (SQLite dev mode)
  2. ensure_all_tables() -- creates all ORM tables if absent
  3. Additive migration: news_articles.content_hash column + index
  4. Recover stale fundamental analysis extraction jobs
  5. Backfill yf_ticker for stocks missing Yahoo Finance symbols
  6. start_scheduler() -- APScheduler daily price + snapshot jobs
  7. Production security audit: warns if SECRET_KEY is default

Custom JSON response class (SafeJSONResponse): serialises NaN/Inf
as null to prevent invalid JSON from financial calculations.

3.4  AUTHENTICATION & SECURITY
-------------------------------
Files: app/core/security.py, app/api/v1/auth.py, app/services/auth_service.py

Auth flow:
  1. POST /api/v1/auth/login -- username + password
  2. Lockout check (failed_login_attempts, locked_until in users table)
  3. bcrypt.checkpw() -- LEGACY_PLAINTEXT_LOGIN auto-upgrades on match
  4. Returns: access_token (JWT, 6h) + refresh_token (JWT, 30d) + user info
  5. POST /api/v1/auth/refresh -- refresh token rotation
     Old refresh token blacklisted via JTI in refresh_tokens table
  6. POST /api/v1/auth/logout -- blacklists refresh token JTI

Token payload:
  Access:  {user_id, username, is_admin, token_type="access", jti, exp}
  Refresh: {user_id, token_type="refresh", jti, exp}

Password reset:
  OTP-based via email (10-min expiry, 5-attempt limit)

Google Sign-In:
  POST /api/v1/auth/google -- id_token validated server-side via Google tokeninfo
  Creates new user or links to existing user by email

Account lockout:
  Configurable: 5 attempts, 15-min lockout
  State stored in users.locked_until (unix timestamp)

Rate limiting:
  SlowAPI rate limits on login endpoints (brute-force prevention)

Admin gating:
  require_admin() dependency checks is_admin=1 in users table

Field encryption:
  Fernet symmetric encryption (FIELD_ENCRYPTION_KEY) for sensitive fields

3.5  MIDDLEWARE STACK
----------------------
File: backend-api/app/core/middleware.py

Middleware registered in order (main.py):
  1. CorrelationIDMiddleware     X-Correlation-ID on every request
  2. RequestTimingMiddleware     X-Response-Time-Ms + structured latency logs
  3. SecurityHeadersMiddleware   OWASP headers:
       X-Content-Type-Options: nosniff
       X-Frame-Options: DENY
       X-XSS-Protection: 1; mode=block
       Referrer-Policy: strict-origin-when-cross-origin
       Content-Security-Policy: default-src 'self'; frame-ancestors 'none'
       Permissions-Policy: interest-cohort=()
       Strict-Transport-Security: max-age=63072000 (prod only)
  4. RequestSizeLimitMiddleware  413 if body > MAX_REQUEST_BODY_BYTES
  5. PrivateNetworkAccessMiddleware  Chrome PNA preflight support
  6. CORSMiddleware              Configurable origins from settings
  7. SlowAPI rate limiter        429 on exceeded limits

Error handlers:
  RateLimitExceeded  -> 429
  APIError subtypes  -> typed 4xx/5xx
  Unhandled          -> 500 with correlation ID

  Symbol resolution: DB yf_ticker -> stock_lists.py -> variation aliases -> suffix rule
  KWD normalisation on fetched prices

fx_service.py
  USD/KWD rate cached for FX_CACHE_TTL seconds
  Fallback to DEFAULT_USD_TO_KWD constant
  convert_to_kwd() used across all monetary outputs

extraction_service.py (Fundamental Analysis PDF pipeline)
  Step 1: PDF -> 250 DPI images (PyMuPDF)
  Step 2: Reasoning pass (Gemini AI scratchpad arithmetic)
  Step 3: Extraction + audit (AI self-validates totals vs. sum-of-parts)
  Step 4: Code-level cross-checks; targeted column retries on mismatch
  Step 5: Content-hash caching (MD5) -- re-upload skips AI calls

ai_service.py
  Google Gemini via google-genai SDK
  Portfolio context assembly (holdings, transactions, performance)
  Supports English and Arabic responses

audit_service.py

push_service.py / notification_prefs.py / portfolio_alerts.py

3.9  CRON / SCHEDULER
----------------------
File: backend-api/app/cron/scheduler.py

APScheduler (BackgroundScheduler):
  Daily job at PRICE_UPDATE_HOUR:PRICE_UPDATE_MINUTE (Asia/Kuwait tz)

  _run_daily_price_then_snapshot():
    1. Fetches DISTINCT user_ids from stocks table
    2. For each user:
         run_price_update(user_id)  [yfinance]
         run_snapshot_save(user_id) [portfolio snapshot in DB]
    3. portfolio_alerts.notify_portfolio_updates_for_users() [Expo Push]

  Cross-worker file lock prevents duplicate runs (multi-process gunicorn)
  Manual trigger: POST /api/v1/cron/update-prices (requires CRON_SECRET_KEY)
  News poller: periodic Boursa Kuwait announcement ingestion

4.4  API CLIENT LAYER
----------------------
File: mobile-app/services/api/client.ts

  Single Axios instance pointed at API_BASE_URL (from constants/Config.ts)
  Request interceptor: attaches Bearer access token from SecureStore
  Client-side rate limiter: 30 req / 10s window
  Response interceptor (reactive 401 handling):
    - refreshInFlight flag prevents concurrent refresh storms (atomic module-level lock)
    - Promise queue: concurrent 401s wait for single refresh, then all retry
    - Circuit breaker: MAX_REFRESH_ATTEMPTS=2 consecutive failures -> forced logout
    - Refresh token rotation: new tokens stored, old tokens cleared
  isAuthEndpoint() guard: skips refresh logic for auth routes

6. DATA FLOW & ARCHITECTURE
================================================================================

Request lifecycle (authenticated API call):
  Mobile App
    | Axios client attaches Bearer token
    v
  FastAPI CORS + Security Middleware
    | Validates origin, sets security headers
    v
  Rate Limiter (SlowAPI)
    | 429 if limit exceeded
    v
  Route Handler
    | get_current_user() dependency: decodes JWT, loads user
    v
  Service Layer
    | Business logic, DB queries, external API calls
    v
  SafeJSONResponse
    | Serialises NaN/Inf as null
    v
  Mobile App (TanStack Query cache)

Daily price update flow:
  APScheduler (14:00 Kuwait)
    -> price_updater.run_price_update(user_id)  [yfinance]
    -> snapshot_saver.run_snapshot_save(user_id) [DB snapshot]
    -> portfolio_alerts.notify_portfolio_updates_for_users() [Expo Push]

Fundamental analysis flow:
  Mobile App -> POST /api/v1/fundamental/upload-pdf
    -> Job created in DB (status=pending)
    -> Background thread: PDF -> images -> Gemini reasoning -> extraction -> verify
    -> Mobile polls: GET /api/v1/fundamental/job/{id}
    -> Results stored in DB + returned to client

Authentication flow:
  Login -> access_token (6h) + refresh_token (30d)
  401 on API call -> Axios interceptor queues requests
    -> POST /auth/refresh -> new access_token + rotated refresh_token
    -> Queued requests retried with new token
  2 consecutive refresh failures -> forced logout + TanStack Query cache clear

State management:
  Server state:  TanStack Query (auto-stale, refetch on focus, MMKV offline persistence)
  Auth state:    Zustand authStore (persisted to SecureStore / localStorage)
  UI state:      Local React state / URL params (Expo Router)
  Theme:         Zustand themeStore (persisted to MMKV)
